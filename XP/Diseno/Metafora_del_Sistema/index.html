Metáfora del Sistema: El Hormiguero Militar AntCiberDron

Concepto guía que unifica arquitectura, actores y comportamiento del sistema

1. Visión General de la Metáfora

Concepto CentralAnalogía de Guerra

El sistema es un hormiguero militar autónomo que coordina múltiples agentes (AntCiberDrones) para interceptar y neutralizar objetivos bélicos.

Así como las hormigas guerreras rusas trabajarían en coordinación, cada AntCiberDron es un agente autónomo que recibe instrucciones de inteligencia desde el Centro de Comando (KGD) y ejecuta misiones de forma independiente pero coordinada.

2. Componentes Principales y sus Analogías

KGD (Centro de Comando)Sistema Central

Rol: Es la "reina" del hormiguero. Centraliza las decisiones, autentica operarios, y mantiene el registro de inteligencia.

Responsabilidades:

  • Autenticación y autorización (control de acceso)
  • Gestión de coordenadas y arsenal (base de datos de inteligencia)
  • Distribución de ordenes a drones
  • Registro y auditoría de operaciones
AntCiberDron (Agente Autónomo)Ejecutor de Misiones

Rol: Es el "soldado autónomo" del hormiguero. Recibe coordenadas, horarios y tipos de arsenal; luego evalúa y ejecuta la misión.

Responsabilidades:

  • Leer y procesar coordenadas de ataque desde el CSV
  • Validar el tipo de arsenal usando el autómata BBA
  • Ejecutar la neutralización del objetivo si es válido
  • Reportar estado de la misión al KGD
Bomba BBA (Autómata)Mecanismo de Validación

Rol: Es el "detector inteligente" integrado en cada dron. Reconoce patrones de lenguaje válidos para el arsenal.

Comportamiento:

  • Si el arsenal coincide con el patrón → DETONACIÓN AUTORIZADA
  • Si no coincide → NEUTRALIZACIÓN RECHAZADA
Archivo de Inteligencia (CSV)Ordenes de Misión

Rol: Es el "parte de guerra" que contiene todas las coordenadas, horarios y tipos de arsenal.

Estructura: Cada fila es una misión (coordenada, día, franja horaria, tipo de arsenal)

Flujo: El KGD lee este archivo → asigna drones → cada dron ejecuta su misión

Panel de Control (UI Hormiguero)Visualización Táctica

Rol: Es el "mapa de comando" donde operarios ven el estado en tiempo real del hormiguero.

Información Visible:

  • Identidad del equipo y operarios autenticados
  • Coordenadas cargadas y procesadas
  • Estado de cada drone (en misión, completado, rechazado)
  • Estadísticas de neutralización
  • Entrenamientos activos en idiomas

3. Flujo de Misión: De la Orden a la Ejecución

Paso 1: Autenticación (Identificación del Operario)

El operario se identifica ante el KGD.

→ Sistema: Valida credenciales y carga el equipo del operario.

Paso 2: Carga de Inteligencia (Lectura del CSV)

El KGD lee el archivo CSV que contiene todas las misiones.

→ Sistema: Muestra una barra de progreso (loading) mientras procesa cada coordenada.

Paso 3: Asignación de Drones (Iteración por Misión)

Para cada coordenada del CSV, el KGD asigna un AntCiberDron.

→ Sistema: Cada dron recibe (geoposición, día, horario, tipo de arsenal).

Paso 4: Validación con Bomba BBA (Evaluación del Arsenal)

El dron envía el tipo de arsenal al autómata BBA para validar si es detonable.

→ Ejemplo: "abcdt" coincide con L={abcdt⁺, ab*} → VÁLIDO ✓

Paso 5: Ejecución (Neutralización del Objetivo)

Si el arsenal es válido, el dron se moviliza a la coordenada y ejecuta la neutralización en el horario especificado.

→ Sistema: Reporta éxito y actualiza estadísticas de misiones completadas.

Paso 6: Entrenamiento (Mejora Continua)

Los operarios entrenan sus habilidades en múltiples idiomas para mejorar comunicación y coordinación.

→ Sistema: Registra progreso y desbloquea niveles de dificultad.
