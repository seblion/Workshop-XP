# Workshop-XP: Proyecto AntCiberDron

Este repositorio sirve como documentación y evidencia de la aplicación de la metodología ágil **Extreme Programming (XP)** para el desarrollo del proyecto **AntCiberDron**.

El objetivo es demostrar la implementación práctica de los valores, principios y prácticas de XP a lo largo del ciclo de vida del software.

## Estructura del Proyecto

La estructura de carpetas refleja las fases y prácticas fundamentales de XP:

### 1. Planificación (Planning)
Ubicación: `/XP/Planificacion`
En esta sección se definen los alcances y tiempos del proyecto.
- **Historias de Usuario**: Requerimientos escritos desde la perspectiva del cliente.
- **Plan de Entregas (Release Plan)**: Cronograma de lanzamientos.
- **Iteraciones**: Planificación de ciclos cortos de desarrollo.
- **Velocidad de Proyecto**: Métrica de trabajo realizado por iteración.
- **Reuniones**: Daily stand-ups y reuniones de planificación.

### 2. Diseño (Design)
Ubicación: `/XP/Diseno`
Enfoque en la simplicidad y claridad del sistema.
- **Metáfora del Sistema**: Visión compartida de cómo funciona el sistema (AntCiberDron).
- **Tarjetas CRC**: Clase-Responsabilidad-Colaborador para diseño de objetos.
- **Soluciones Puntuales (Spike Solutions)**: Experimentos para reducir riesgos técnicos.
- **Funcionalidad Mínima**: Principio de "Simple Design".
- **Reciclaje (Refactoring)**: Mejora continua del código sin cambiar su comportamiento.

### 3. Desarrollo (Coding)
Ubicación: `/XP/Desarrollo`
El núcleo de la construcción del software.
- **Programación por Parejas**: Dos desarrolladores, una máquina.
- **Unidad de Pruebas (Unit Tests)**: Pruebas automatizadas a nivel de código (TDD).
- **Integración**: Integración continua del código.
- **Disponibilidad del Cliente**: Interacción constante con el cliente real.

### 4. Pruebas (Testing)
Ubicación: `/XP/Pruebas`
Garantía de calidad y validación.
- **Pruebas de Aceptación**: Pruebas funcionales definidas por el cliente.
- **Implantación**: Despliegue del sistema.
